# Editorial Style Wiki

## What works (keep doing)
- Lead with the single best confirmed update up front; a direct return/decision note makes the piece immediately usable.
- Keep the headline, subhead, and lede tightly aligned on the same core development for a clean breaking-news read.
- Stay strictly inside the source digest when the item is only a headline-level claim; concise beats embellished.
- Use concrete timing or status language when it is directly supported by the source.
- A narrow, source-bound update can carry the story without extra context or broader trend framing.
- Let verified facts do the work instead of trying to add rotation, market, or strategy implications the source does not provide.
- Strong grounding can offset a simple setup when the execution stays clean and precise.
- [insider] Build around the one supported update, not a layered explainer or team-context add-on.

## What to avoid
- Don’t soften a clear source claim with hedging, then overstate it with extra narrative around it.
- Don’t add rotation, pass-rush, revamp, or broader context framing unless the digest explicitly supports it.
- Don’t let banned source-narration/meta wording creep into the intro.
- Don’t make the headline, subhead, and lede pull in different directions; keep the first read unified.
- Don’t widen a single confirmed update into a broader team-effect story without evidence.
- Don’t pad a short source-backed note with unsupported implications or next-step speculation.
- Don’t treat a headline-level digest like a full-reporting source.
- [insider] Don’t build beyond the one supported update into rotation or scheme interpretation unless the source gives that basis.
- [source-side, not fixable in prose] If the digest only provides a thin headline claim or empty digest set, the fix is more source detail, not heavier prose.
- [source-side, not fixable in prose] Don’t invent player or team context when the digest supplies none; unsupported names and comparisons are a source gap, not an execution fix.
