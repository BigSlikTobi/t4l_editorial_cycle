# Editorial Style Wiki

## What works (keep doing)
- Lead with the single verified update up front; a direct status/decision note makes the piece usable fast.
- Keep headline, subhead, and lede aligned on the same core development for a clean first read.
- Stay strictly inside the source digest when the item is thin or headline-level; concise beats embellished.
- Use exact status language when it is clearly supported by the source.
- A narrow, source-bound update can carry the story without extra context or trend framing.
- Let verified facts do the work instead of adding implications the source does not provide.
- Strong grounding can offset a simple setup when the execution stays clean and precise.
- [analyst] Keep a betting/market lede tied to the verified opening claim; if the digest is thin, precision matters more than explanation.
- [insider] Build around the one supported update, not a layered explainer or context add-on.

## What to avoid
- Don’t soften a clear source claim with hedging, then overstate it with extra narrative around it.
- Don’t add broader team, market, scheme, or strategy framing unless the digest explicitly supports it.
- Don’t let banned source-narration/meta wording creep into the intro.
- Don’t make the headline, subhead, and lede pull in different directions; keep the first read unified.
- Don’t widen a single confirmed update into a broader story without evidence.
- Don’t pad a short source-backed note with unsupported implications or next-step speculation.
- Don’t treat a headline-level digest like a full-reporting source.
- Don’t repeat the same claim across headline, subhead, and intro when the digest is empty; repetition doesn’t add substance.
- [analyst] Don’t turn an early-favorites story into market analysis or team-specific implication unless the digest actually provides that basis.
- [insider] Don’t build beyond the one supported update into rotation, scheme, or context interpretation unless the source gives that basis.
- [source-side, not fixable in prose] If the digest is empty or only provides a thin claim, the fix is more source detail, not heavier prose.
- [source-side, not fixable in prose] Don’t invent player or team context when the digest supplies none; unsupported names and comparisons are a source gap, not an execution fix.
- [source-side, not fixable in prose] If there are no source facts at all, even a clean headline claim can’t be supported in prose.
